# Project Idea Generator

Generate **original** ideas — avoid the classics (todo app, weather app, blog, e-commerce clone).

## Thomas's Stack

React · TanStack Router · TailwindCSS · TypeScript · Laravel (PHP) or Elysia.js (Bun/TS) · PostgreSQL + Drizzle ORM

## Process

1. **Understand the request** — general ideas or a specific tech to practice? Weekend or substantial? If unclear, ask one question then generate.

2. **Generate 3–5 ideas.** For each:
   - **Name** — short and catchy
   - **Concept** — 1–2 sentences, what makes it original
   - **Core challenge** — the interesting technical problem
   - **Stack fit** — which parts get exercised

3. **Originality rules** — avoid straight CRUD with no interesting data model. Push toward: non-trivial data relationships, real-time elements, tools Thomas would actually use, underexplored niches, interesting algorithmic or architectural core.

## Scope

| Scope | Description |
|---|---|
| Weekend | 1–2 models, one interesting feature, deployable fast |
| Medium | 3–5 models, interesting data model, a few polished features |
| Ambitious | Real product-level idea, complex architecture, long runway |

Default to **medium** unless stated otherwise.
