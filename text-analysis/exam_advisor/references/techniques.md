# Revision Techniques by Exam Type

## Essay / Long-answer

1. **Recall dump** — without notes, write everything you know in 5 min
2. **Check** — open notes, see what you missed
3. **Targeted re-read** — only the sections covering gaps
4. **Timed answer** — full answer under exam conditions
5. **Mark it** — against mark scheme or ask an AI to evaluate against the criteria

Repeat per key topic. Do at least one full timed mock.

## Multiple Choice / Short Answer

- Flashcards (Anki or paper deck)
- Practice question banks — mark immediately after each question
- Review wrong answers first: understand *why*, not just what's right
- Cluster by topic to identify weak areas

## Problem-solving / Calculation

1. Study one worked example until each step is clear
2. Cover the solution and re-do it
3. Do a new problem of the same type without help
4. Diagnose which step failed if wrong
5. Move on only after 3 in a row correct

## Practical / Implementation

- 20-min timer per concept; look up docs only when completely stuck
- Keep a "gap log" of things needed to look up
- Review the gap log the night before

# Techniques by Concept Type

## Theoretical (laws, models, frameworks)

- Feynman technique — explain out loud to an imaginary person
- Draw a diagram or mind map from memory
- Connect to a real-world example — examiners reward application

## Procedural / step-by-step

- Write the steps from memory, in order
- Identify the step you always forget — make a mnemonic
- Practise end-to-end, not just individual steps

## Comparative / analytical

- Comparison table from memory
- State judgement first ("X is more effective because…"), then support it
- Write two counter-arguments and refute them — this is what top-band answers do

## Coding / technical implementation

- Code without autocomplete on first pass
- Use docs only when completely stuck — note what you looked up
- Write a short comment explaining *why* each major section works
- Test edge cases, not just the happy path

# Quick Reference

| Technique | Best for | Time cost |
|---|---|---|
| Active recall / flashcards | Facts, definitions, terminology | Low |
| Timed past paper answers | Essays, long-answer | High |
| Worked example re-doing | Calculation, problem-solving | Medium |
| Feynman technique | Complex concepts | Low–Medium |
| Spaced repetition (Anki) | Any factual content over 2+ weeks | Low per session |
| Mind mapping from memory | Checking breadth of knowledge | Low |
| Build from scratch | Practical / coding / implementation | High |
| Peer teaching | Any topic — exposes gaps | Medium |
